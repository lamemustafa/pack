# Product

## Register

product

## Users

Indian compliance operators, accountants, and CAs using an already-authenticated GST Portal browser session to collect filed-return artifacts locally. They are often working through month or full-year batches, need clear evidence of what happened, and cannot risk wrong-taxpayer or wrong-period downloads.

## Product Purpose

ComplyEaze Pack is a local-first browser extension that turns explicit GST Portal download actions into target-bound local files while preserving a no-login, no-cookie, no-upload trust boundary. Success means the user can choose a return type, period, and artifact set, run it with minimal repeated browser prompts, and understand every terminal state without exposing sensitive taxpayer data.

## Brand Personality

Calm, precise, private. The product should feel like a careful compliance tool: operational, compact, and trustworthy rather than decorative or promotional.

## Anti-references

Do not make the extension feel like a marketing landing page, a dashboard with decorative cards, a flashy SaaS control panel, or a generic AI-generated settings screen. Avoid oversized hero treatment, decorative gradients, vague success copy, hidden failure states, and UI that buries the actual download target.

## Design Principles

- Target first: return type, financial year, period, artifact, and run state must be visible before action.
- Evidence without exposure: show safe path/status signals, never raw URLs, filenames, taxpayer IDs, portal HTML, or artifact bytes.
- Batch friction belongs at the end: where possible, consolidate multi-file work into one explicit ZIP handoff.
- Operational density: use restrained layout, direct controls, and clear terminal states over explanatory prose.
- Recover honestly: blocked, unsupported, cancelled, and retryable states must be distinct and actionable.

## Accessibility & Inclusion

Use familiar form controls and buttons, keyboard-accessible actions, visible focus states, readable contrast, non-color-only status cues, and reduced-motion-safe transitions. Default target is WCAG 2.1 AA for the extension popup/options surfaces.
